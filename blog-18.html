DevOps & CI/CD

DevOps Last updated: April 2024

What is DevOps?

Combination of development and operations culture focused on automating the software delivery pipeline for faster, reliable releases.

Core Principles

  • Collaboration between dev and ops teams
  • Automation of repetitive tasks
  • Continuous improvement
  • Monitoring and feedback

CI/CD Pipeline

Continuous Integration (CI)

Automatically build and test code on every commit:

  • Code commit triggers automated build
  • Run unit tests and integration tests
  • Code quality analysis
  • Build artifact creation

Continuous Deployment (CD)

Automatically deploy tested code to production:

  • Staging environment testing
  • Automated or manual approval
  • Production deployment
  • Health checks and rollback capability

Popular CI/CD Tools

  • Jenkins: Open-source automation server
  • GitLab CI: Built into GitLab
  • GitHub Actions: GitHub native CI/CD
  • CircleCI: Cloud-based CI/CD
  • Travis CI: Cloud-based for GitHub

Infrastructure as Code (IaC)

Define infrastructure using code for version control and repeatability:

  • Terraform: Provision cloud resources
  • Ansible: Configuration management
  • CloudFormation: AWS infrastructure templates
  • Docker: Containerization

Containerization

Docker: Package application and dependencies in containers

Dockerfile: Define container image

Docker Compose: Multi-container orchestration

Container Orchestration

Kubernetes: Automate deployment, scaling of containers

  • Pod management
  • Service discovery
  • Load balancing
  • Self-healing and auto-scaling

Monitoring and Logging

  • Prometheus: Metrics collection
  • ELK Stack: Elasticsearch, Logstash, Kibana
  • Grafana: Visualization and dashboards
  • DataDog: Comprehensive monitoring

Version Control

Git: Distributed version control

Branching Strategies: Git Flow, GitHub Flow

Remember: DevOps is about culture, process, and tools - prioritize in that order.
