Blog #12

Cloud Computing

Infrastructure Last updated: April 2024

What is Cloud Computing?

On-demand access to computing resources (servers, storage, databases) over the internet with pay-as-you-go pricing.

Service Models

IaaS (Infrastructure as a Service)

Rent virtualized computing resources over internet. Examples: AWS EC2, Azure Virtual Machines

PaaS (Platform as a Service)

Platform for developing, testing, and deploying applications. Examples: Heroku, Google App Engine

SaaS (Software as a Service)

Ready-to-use applications accessed via web browser. Examples: Gmail, Office 365, Salesforce

Deployment Models

  • Public Cloud: Multi-tenant, shared resources
  • Private Cloud: Dedicated for single organization
  • Hybrid Cloud: Mix of public and private
  • Community Cloud: Shared by multiple organizations

Major Cloud Providers

  • AWS (Amazon Web Services): Largest market share
  • Microsoft Azure: Enterprise integration
  • Google Cloud Platform: Data analytics focus
  • IBM Cloud, Oracle Cloud, Alibaba Cloud

Containerization (Docker)

Package applications with dependencies into containers for consistency across environments.

Orchestration (Kubernetes)

Automate deployment, scaling, and management of containerized applications.

Serverless Computing

Execute code without managing servers. Examples: AWS Lambda, Azure Functions

Benefits of Cloud

  • Cost savings (no infrastructure investment)
  • Scalability and flexibility
  • High availability and disaster recovery
  • Security and compliance
  • Easy updates and maintenance
Remember: Choose cloud services based on your specific application needs and budget.
