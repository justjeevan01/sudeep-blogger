Object Oriented Programming

Programming Last updated: April 2024

What is OOP?

Object Oriented Programming (OOP) is a programming paradigm based on organizing code into objects, which contain data and methods.

Four Pillars of OOP

1. Encapsulation

Bundling data (variables) and methods together, hiding internal details and exposing only what's necessary.

2. Inheritance

Creating a new class based on an existing class. Promotes code reusability and establishes relationships.

3. Polymorphism

Objects can take multiple forms. Method overloading and method overriding are examples.

4. Abstraction

Hiding complexity and showing only essential features. Abstract classes and interfaces achieve this.

Class and Object

Class: A blueprint for creating objects

Object: An instance of a class

Access Modifiers

  • Private: Accessible only within the class
  • Protected: Accessible within the class and subclasses
  • Public: Accessible from anywhere

Key Concepts

Constructor: Initializes object state

Destructor: Cleans up resources when object is destroyed

Static members: Shared across all instances

this keyword: Refers to current object

Remember: Design classes to model real-world entities for better understanding.
